# Ответ Яндекса по Fleet: роли, ключи, распределение водителей

Краткая выжимка официального ответа (для контекста продукта и безопасности).

---

## Что есть в Fleet

- **Директор** может добавлять пользователей и настраивать **роли** с разными названиями и правами доступа.
- Можно сделать **индивидуальные настройки по API-ключам для каждой роли** и выдать ключи менеджерам.

## Чего нет в Fleet

- **Нет функции распределения водителей** или их группировки с последующим закреплением за конкретным менеджером. То есть «привязать водителей к менеджеру» на стороне Yandex нельзя — логику «кто какими водителями управляет» реализует партнёр (мы) у себя в приложении.

## Ответственность за ключи

- **Ответственность за сохранность данных лежит только на партнёре.** Партнёр обязан убедиться, что ключи не попадут в руки злоумышленников.

---

**Вывод для нашего приложения:** модель «каждый менеджер вводит свой API-ключ в онбординге, ключ хранится в нашей БД и используется для запросов к Fleet от его имени» — соответствует тому, что описал Яндекс (ключи на роль/менеджера, безопасность на стороне партнёра). Распределение водителей по менеджерам мы делаем сами (например через DriverLink и привязку по телефону в нашем кабинете).
